# 手動バックアップ試験 事前準備ガイド（MBK-013〜015）

## 目的
- 手動バックアップの大量データ試験（MBK-013〜015）を再現可能に実施する。
- 1000件データを手作業で作らず、デバッグ用Seedで一度生成して `.esbackup` フィクスチャ化する。

## 事前前提
- Pro状態で試験する（無料プランは50件上限）。
- `FORCE_RC_FALLBACK=1` を使っていた場合は解除する。
- `ENABLE_BACKUP_PAYWALL` は既定どおり有効のままでよい。
- パスフレーズは8文字以上。
- インポート対象は `.esbackup` のみ。

## 1000件データ生成（デバッグSeed）
1. Xcode Scheme の `Environment Variables` に `SEED_BULK_EPISODES=1000` を設定する。  
2. iCloud同期をオフにした状態でアプリを起動する。  
3. エピソード一覧で1000件作成されていることを確認する。  
4. 1回生成できたら `SEED_BULK_EPISODES` を解除する。  

補足:
- `-SEED_MANUAL_BACKUP_FIXTURE` の起動引数でも1000件生成できる。
- Seedは既存データが空のときのみ実行される。
- iCloud同期が有効な場合はSeedをスキップする。

## フィクスチャ運用
1. 1000件状態で「バックアップをダウンロード」を実行し `.esbackup` を保存する。  
2. 以後のMBK-014/015は、このファイルを使い回して計測する。  
3. 計測は同一端末・同一ビルドで実施する。  
4. Instruments は `Time Profiler` と `Allocations` を使用する。  

## 記録時の注意
- 失敗時は端末状態差分（iCloud同期ON/OFF、ビルド差、課金状態差）を実結果欄に残す。
- 目標値は export/import 各60秒以内、ピークメモリ500MB未満。
